## 主要決策

### 決策 1：bypass 分支用 cf_smart_cache_record_bypass 而非 cf_smart_cache_increment_miss
- **原因**：兩個函數會重複計數（共用 transient key）
- **取捨**：保留兩個函數，bypass 分支只用 record_bypass；miss 計數保留供未來手動呼叫

### 決策 2：cacheable 分支 URL 用 home_url(add_query_arg(array(), $wp->request))
- **原因**：home_url() 回傳完整 URL 含 scheme/host；$wp->request 已經過 WP 解析
- **影響**：URL 記錄與 Cloudflare 看到的一致

### 決策 3：admin.php 採用表格而非圖表
- **原因**：零外部依賴、不引入 Chart.js、避免改動 WordPress 後台風格

### 決策 4：使用 Node.js 讀寫檔案而非 PowerShell
- **原因**：Set-Content 會把 CRLF 改成 LF，PowerShell 還會把 markdown 中的 ** 當作運算子解析
- **影響**：保留原編碼、避免 here-string 解析錯誤
